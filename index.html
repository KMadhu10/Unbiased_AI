How to make it unbiased

Making an AI model unbiased is a three-stage process — you fix problems before training, during training, and after training. Think of it like baking a cake: you need the right ingredients, the right recipe, and the right finishing touches. Miss any stage and the result is still flawed.

Stage 1 — Before training
Fix your data first

Bad data in = bad decisions out. If your training data shows that women were historically rejected more, the AI will learn to keep doing that. You need to clean this up first.

  • Remove proxy features — columns like zip code or school name that secretly carry race or gender information
  • Balance the dataset — add more examples of underrepresented groups (e.g. female applicants who were hired)
  • Check historical data — data from before 2021 in this case reflects old biases; down-weight or exclude it
Stage 2 — During training
Teach the model to be fair

When the AI is learning patterns, you can add a "fairness rule" that penalises it every time it treats groups differently without a good reason — just like a teacher marking down a student for bias.

  • Add a fairness penalty to the training loss so the model is punished for creating large approval rate gaps
  • Use adversarial debiasing — train a second model that tries to predict gender/race from decisions; if it can, the first model is still biased
  • Apply constraint-based learning to enforce equal true positive rates across groups
Stage 3 — After training
Adjust the final decisions

Even after a fair model is trained, the cut-off point for a "yes" or "no" decision can still create gaps. You can tune this separately for each group to level the playing field at the output stage.

  • Apply per-group thresholds — slightly lower the bar for historically disadvantaged groups to correct for systematic underscoring
  • Use calibration to ensure that a 70% confidence score means the same thing regardless of the applicant's background
  • Monitor in production — bias can drift back over time as new data comes in; audit quarterly
What the ideal outcome looks like

The goal is not to make the AI approve everyone equally — it is to make sure that equally qualified people have equal chances, regardless of gender, race, age, or any other protected characteristic. Here is what success looks like:

≥ 80
Target fairness score — the model is treating all groups fairly by international standards
< 5%
Maximum demographic parity gap — approval rate differences between groups should be under 5 percentage points
> 0.80
80% rule ratio — for every 100 men approved, at least 80 women with similar profiles should also be approved
0

Remediation plan
Evidence-based interventions to reduce bias before deployment, ordered by projected impact.
1
Remove proxy variables (pre-processing)
Drop zip code, school name, and surname from the feature set. These three features alone explain 38% of the demographic parity gap. Replace with geodemographic quintile scores that are race-neutral by construction.
−14.2pp gap reduction
2
Resample training data (pre-processing)
Apply 1.5× SMOTE oversampling on underrepresented groups for pre-2021 data. This corrects the historical imbalance without discarding records. Combined with temporal weighting for recent data.
−7.8pp gap reduction
3
Add fairness regularization (in-processing)
Incorporate a fairness penalty term (λ=0.1) into the model's loss function using the Reduction approach. This constrains the model to equalize true positive rates across groups during training.
−9.1pp gap reduction
4
Per-group threshold calibration (post-processing)
Apply group-specific decision thresholds to equalize false positive rates. This intervention preserves model accuracy while mechanically enforcing equalized odds at decision time. Requires ongoing monitoring.
−5.3pp gap reduction
5
Continuous monitoring & audit cadence
Deploy FairSight's real-time monitoring dashboard. Set alerts for demographic parity drift >3pp. Schedule quarterly full audits. Document all decisions for the EU AI Act Art.13 transparency log.
